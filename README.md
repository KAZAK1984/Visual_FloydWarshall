# Visual Floyd-Warshall

Программный комплекс для интерактивной визуализации алгоритма Флойда-Уоршелла, разработанный в рамках курсовой работы первого курса по дисциплине «Объектно-ориентированное программирование» (УлГТУ, ФИСТ).

## 📌 Обзор проекта

Программа представляет собой десктопное приложение на базе **WPF**, предназначенное для демонстрации работы динамического алгоритма поиска кратчайших путей между всеми парами вершин взвешенного графа. Основной акцент сделан на пошаговом разборе каждой итерации и возможности сохранения результатов вычислений для последующего анализа.
## 🏗 Архитектура и ООП

Проект спроектирован с соблюдением принципов SOLID и акцентом на расширяемость:

* **Абстракция конфигураций:** Используется иерархия классов `AlgorithmConfig` и `FloydAlgorithmConfig` для инкапсуляции входных данных (матрица смежности, параметры коллекции).
* **Интерфейсы и слабая связность:** Основная логика вычислений и управления скрыта за интерфейсом `IAlgorithmRunner`, что позволяет легко подменять алгоритмы или механизмы работы с файлами.
* **Логирование:** Реализована система журналирования через интерфейс `ILogger` с потокобезопасной записью в файл (`FileLogger`).
* **Фабрики рендеринга:** Для отрисовки графа и матрицы расстояний используются специализированные фабрики (`GraphRenderingFactory`, `DistanceMatrixRenderingFactory`), что отделяет визуальное представление от бизнес-логики.

## 🚀 Ключевые возможности

1.  **Интерактивная визуализация:**  Пошаговый выбор итераций (k, i, j) через слайдеры или автоматический переход.
    * Графическое отображение текущей проверки пути: оранжевый узел — промежуточный (k), красный и зеленый — пара (i, j).
    * Подсветка формулы улучшения расстояния: `D[i,j] = min(D[i,j], D[i,k] + D[k,j])`.
2.  **Работа с данными:**
    * Генерация графа по заданному количеству вершин (от 4 до 40).
    * Сохранение и загрузка состояния в кастомный формат файлов `.floyd` с проверкой сигнатуры `VFLOYD`.
3.  **Диагностика:**
    * Обнаружение отрицательных циклов.
    * Ведение подробного лога событий приложения в `visual_floyd.log`.

## 📂 Структура проекта

* `/Algorithm` — ядро: решатель, интерфейсы и модели данных.
* `/Logging` — компоненты системы логирования.
* `/Rendering` — логика графической отрисовки элементов Canvas.
* `/UI` — описание интерфейса (XAML) и логика управления формой.
* `/Utility` — вспомогательные конвертеры и расширения для WPF.
